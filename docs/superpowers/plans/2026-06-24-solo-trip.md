# 独自出发功能 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在匹配页加"独自出发"按钮，允许用户跳过组队直接进入旅行页，并在旅行页适配无搭子的独行模式。

**Architecture:** match.vue 底部新增"独自出发"按钮，点击携带 `solo=true` 跳转 trip.vue；trip.vue 读取该参数后，用 `isSolo` ref 守卫所有 WebSocket 调用并调整 UI（隐藏搭子行、过滤搭子挑战）。

**Tech Stack:** UniApp + Vue 3 Composition API + TypeScript，微信小程序平台

## Global Constraints

- 不新增任何后端接口，纯前端改动
- `solo=true` 通过 URL query 传递，`trip.vue` 用 `query?.solo === 'true'` 读取
- 所有 `sendMatch` / `disconnectMatch` 调用须用 `if (!isSolo.value)` 守卫，不得删除原有调用逻辑
- "独自出发"按钮样式为次要（secondary），"开始匹配"保持主要（primary）样式
- 两按钮均在 `!selectedSpot` 时禁用
- `visibleChallenges` 过滤掉 id 为 `near_partner` 和 `partner_active` 的挑战

## Task 1: match.vue — 新增"独自出发"按钮

**Files:**

**Interfaces:**
- Produces: 点击"独自出发"后执行 `goSolo()`，跳转到 `/pages/trip/trip?spotId=X&spotName=Y&solo=true`

- [ ] **Step 1: 在 `<script setup>` 中添加 `goSolo()` 函数**

在 `match.vue` 的 `<script setup>` 中，在 `startMatch()` 函数之后添加：

```typescript
function goSolo() {
  if (!selectedSpot.value) return
  uni.redirectTo({
    url: `/pages/trip/trip?spotId=${selectedSpot.value.id}&spotName=${encodeURIComponent(selectedSpot.value.name)}&solo=true`
  })
}
```

- [ ] **Step 2: 修改 template 中的 `bottom-bar`**

找到现有的 `bottom-bar`：

```html
<view class="bottom-bar">
```

```html
<view class="bottom-bar">
```

## Task 2: trip.vue — solo 模式适配

**Files:**
- Modify: `frontend/pages/trip/trip.vue`

**Interfaces:**
- Consumes: URL param `solo=true`（由 Task 1 的 `goSolo()` 传入）
- Produces: `isSolo` ref，守卫所有 WebSocket 调用；`visibleChallenges` computed，过滤搭子挑战

- [ ] **Step 1: 在 `onLoad` 中读取 `solo` 参数**

找到现有的 `onLoad`：

```typescript
onLoad((query) => {
  spotId.value = Number(query?.spotId ?? 0)
  spotName.value = decodeURIComponent(query?.spotName ?? '')
  partnerNickname.value = decodeURIComponent(query?.partnerNickname ?? '搭子')
})
```

替换为：

```typescript
const isSolo = ref(false)

onLoad((query) => {
  spotId.value = Number(query?.spotId ?? 0)
  spotName.value = decodeURIComponent(query?.spotName ?? '')
  partnerNickname.value = decodeURIComponent(query?.partnerNickname ?? '搭子')
  isSolo.value = query?.solo === 'true'
})
```

注意：`isSolo` 的声明放在 `onLoad` 调用之前（与其他 `ref` 声明处同级）。

- [ ] **Step 2: 添加 `visibleChallenges` computed**

在 `completedCount` computed 之后添加：

```typescript
const visibleChallenges = computed(() =>
  isSolo.value
    ? challenges.value.filter(c => !['near_partner', 'partner_active'].includes(c.id))
    : challenges.value
)
```

- [ ] **Step 3: 守卫 `onMounted` 中的 `setMessageHandler`**

找到 `onMounted` 中：

```typescript
onMounted(() => {
  setMessageHandler(onWsMessage)
  updateMyLocation()
  locationTimer = setInterval(updateMyLocation, 3000)
  // ...
})
```

将 `setMessageHandler(onWsMessage)` 改为：

```typescript
onMounted(() => {
  if (!isSolo.value) setMessageHandler(onWsMessage)
  updateMyLocation()
  locationTimer = setInterval(updateMyLocation, 3000)
  // ...
})
```

- [ ] **Step 4: 守卫 `onUnmounted` 中的 `disconnectMatch`**

找到：

```typescript
onUnmounted(() => {
  if (locationTimer) clearInterval(locationTimer)
  if (regionPollTimer) clearInterval(regionPollTimer)
  if (popupDismissTimer) clearTimeout(popupDismissTimer)
  disconnectMatch()
})
```

替换为：

```typescript
onUnmounted(() => {
  if (locationTimer) clearInterval(locationTimer)
  if (regionPollTimer) clearInterval(regionPollTimer)
  if (popupDismissTimer) clearTimeout(popupDismissTimer)
  if (!isSolo.value) disconnectMatch()
})
```

- [ ] **Step 5: 守卫 `updateMyLocation` 中的位置广播**

找到 `updateMyLocation` 中：

```typescript
sendMatch('location', { latitude: res.latitude, longitude: res.longitude })
```

替换为：

```typescript
if (!isSolo.value) sendMatch('location', { latitude: res.latitude, longitude: res.longitude })
```

- [ ] **Step 6: 守卫涂鸦相关的 `sendMatch` 调用**

共 3 处：

**`onDrawEnd` 中：**
```typescript
sendMatch('drawStroke', { id: stroke.id, points: stroke.points })
```
替换为：
```typescript
if (!isSolo.value) sendMatch('drawStroke', { id: stroke.id, points: stroke.points })
```

**`eraseNear` 中：**
```typescript
sendMatch('eraseStroke', { id: stroke.id })
```

- [ ] **Step 7: 守卫 `leaveTrip` 中的 WebSocket 调用**

找到 `leaveTrip` 中：

```typescript
if (res.confirm) {
  sendMatch('leave')
  disconnectMatch()
  uni.redirectTo({ url: '/pages/index/index' })
}
```

替换为：

```typescript
if (res.confirm) {
  if (!isSolo.value) {
    sendMatch('leave')
    disconnectMatch()
  }
  uni.redirectTo({ url: '/pages/index/index' })
}
```

- [ ] **Step 8: 修改 template — 面板搭子行和距离行**

找到面板中的"搭子"和"与搭子距离"两行：

```html
<view class="panel-row">
  <text class="panel-label">搭子</text>
  <text class="panel-value">{{ partnerNickname }}</text>
</view>
<view class="panel-row">
  <text class="panel-label">与搭子距离</text>
```

替换为：

```html

  <text class="panel-label">与搭子距离</text>
  <text class="panel-value">{{ distanceText }}</text>
</view>
```

- [ ] **Step 9: 修改 template — 挑战任务列表改用 `visibleChallenges`**

找到挑战任务区的进度条和列表，将所有 `challenges` 改为 `visibleChallenges`：

进度条（`challenge-progress-fill` 的 style）：
```html
:style="{ width: (completedCount / challenges.length * 100) + '%' }"
```
替换为：
```html
:style="{ width: (completedCount / visibleChallenges.length * 100) + '%' }"
```

挑战摘要文字：
```html

```
替换为：
```html

```

挑战列表 `v-for`：
```html

```
替换为：
```html

```

同时修改 `completedCount` computed，使其也基于 `visibleChallenges`：

找到：
```typescript
const completedCount = computed(() => challenges.value.filter(c => c.completed).length)
```
替换为：
```typescript
const completedCount = computed(() => visibleChallenges.value.filter(c => c.completed).length)
```
