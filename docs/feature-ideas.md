# TripMate 创新功能构想

> 记录日期：2026-06-07

---

## 核心概念：旅行 RPG 化

将旅行体验游戏化——景点是关卡，打卡是战斗，成就是勋章，朋友是队友。

---

## 一、任务系统

用户在旅行中可以接受和完成任务，类似游戏里的「主线 / 支线任务」。

### 任务类型（设想）

| 类型 | 示例 |
|------|------|
| 打卡任务 | 扫描景点二维码完成打卡 |
| 探索任务 | 在同一城市打卡 3 个景点 |
| 时间任务 | 在日出/日落时段打卡 |
| 收集任务 | 集齐某条路线的所有景点 |
| 社交任务 | 和朋友同时在同一景点打卡 |

### 任务属性
- 标题、描述、图标
- 完成条件（类型 + 目标值）
- 奖励积分
- 是否限时

---

## 二、成就系统（Steam 风格）

完成特定条件自动触发成就解锁，弹出通知卡片（类似 Steam 右下角弹窗）。

### 成就示例

| 成就名 | 解锁条件 |
|--------|---------|
| 初出茅庐 | 完成第一次打卡 |
| 美食猎人 | 在 5 个不同城市打卡 |
| 夜枭 | 在晚上 10 点后完成打卡 |
| 探险家 | 解锁 10 个冷门景点 |
| 旅行达人 | 累计打卡 50 个景点 |
| 好友同行 | 和朋友同时在同一景点打卡 |
| 城市征服者 | 完成某城市的全部任务 |

### 成就展示
- 个人主页展示已解锁徽章墙
- 未解锁成就显示为灰色剪影（有神秘感）
- 每个成就有解锁时间记录

---

## 三、扫码导览（AR 导游）

扫描景点二维码 → 自动播放语音讲解 + 显示图文介绍卡片。

### 流程
1. 用户点击「扫一扫」
2. 扫描景点专属二维码
3. 跳转景点详情页：封面图、名称、简介
4. 自动朗读讲解文字（TTS）
5. 同时触发「打卡」逻辑，计入任务/成就进度

### 管理后台支持
- 管理员录入景点信息（名称、图片、讲解文字）
- 自动生成景点二维码，可下载打印

---

## 四、协作足迹地图（多人联机）

和朋友组队，共享一张实时地图，记录双方的旅行轨迹。

### 功能点
- 创建「旅行房间」，邀请好友加入
- 地图上实时显示每位成员当前位置
- 每次打卡在地图上留下足迹标记
- 旅行结束后生成共同足迹回顾图
- 房间内简单聊天/留言

### 技术方向
- WebSocket 实时位置同步
- 腾讯地图 / 高德地图 SDK

---

## 五、旅行副本（组队挑战）

朋友组队前往同一城市，系统为每人分配不同支线任务，全员完成后解锁城市「通关奖励」。

- 强制分工合作，增加互动
- 通关奖励：专属纪念卡、特殊成就徽章

---

## 六、旅行日志自动生成

旅行结束后，根据打卡记录 + 上传照片，自动生成图文旅行报告。

- 时间线格式展示每个打卡点
- AI 润色一段旅行总结文案
- 一键生成图片分享到朋友圈

---

## 实现优先级建议

| 功能 | 优先级 | 难度 | 依赖 |
|------|--------|------|------|
| 景点打卡 + 扫码导览 | ⭐⭐⭐ 高 | 低 | 景点数据表 |
| 任务系统 | ⭐⭐⭐ 高 | 中 | 打卡系统 |
| 成就系统 | ⭐⭐⭐ 高 | 中 | 任务系统 |
| 成就展示页 | ⭐⭐ 中 | 低 | 成就系统 |
| 旅行足迹地图（单人） | ⭐⭐ 中 | 中 | 打卡系统 |
| 旅行日志生成 | ⭐⭐ 中 | 中 | 打卡 + 照片 |
| 协作足迹地图（多人） | ⭐ 低 | 高 | WebSocket |
| 旅行副本 | ⭐ 低 | 高 | 多人系统 |

---

## 最小可行版本（MVP）

只做这三件事就已经是一个完整的差异化产品：

1. **扫码打卡** — 扫景点二维码，完成打卡记录
2. **任务列表** — 展示可完成的旅行任务，进度实时更新
3. **成就解锁** — 满足条件自动触发，弹出 Steam 风格通知
