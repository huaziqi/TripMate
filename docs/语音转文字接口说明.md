# 语音转文字（ASR）接口说明

基于 **科大讯飞语音听写（IAT）** 实现的语音转文字接口。客户端上传一段完整的录音文件，后端通过讯飞 WebSocket 协议完成识别后，一次性返回识别文本。

- 讯飞能力：语音听写（中英文，≤ 60 秒短音频）
- 官方协议文档：<https://www.xfyun.cn/doc/asr/voicedictation/API.html>

---

## 1. 接口定义

| 项目 | 说明 |
|------|------|
| 接口地址 | `POST /api/asr/recognize` |
| 请求类型 | `multipart/form-data` |
| 鉴权 | 无需登录（已在 SecurityConfig 中放行 `/api/asr/**`） |

### 请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `file` | File | 是 | — | 音频文件，支持 `pcm` / `wav` / `mp3` |
| `format` | String | 否 | 按文件扩展名推断 | 音频格式：`pcm` \| `wav` \| `mp3`；扩展名无法推断时按 `pcm` 处理 |
| `language` | String | 否 | `zh_cn` | 识别语种：`zh_cn`（中文，含简单英文）\| `en_us`（英文） |

### 音频要求

| 项目 | 要求 |
|------|------|
| 采样率 | **16000 Hz**（16k） |
| 位深 | 16 bit |
| 声道 | 单声道（mono） |
| 时长 | ≤ 60 秒（讯飞听写上限） |
| 大小 | ≤ 5 MB（服务端 multipart 上限） |

> `wav` 文件会自动去掉文件头后按裸 pcm 上传；`mp3` 按讯飞 `lame` 编码上传。
> 采样率不是 16k 的音频会导致识别结果错乱，请在录音端固定 16000 采样率。

---

## 2. 响应格式

统一响应体 `Result<T>`：`{ code, message, data }`，`code = 200` 表示成功。

### 成功示例

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "text": "今天天气怎么样？",
    "sid": "iat000e2f45@dx1a2b3c4d5e6f7g8h00"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.text` | String | 识别出的文本（含标点） |
| `data.sid` | String | 讯飞会话 ID，联系讯飞排查问题时提供 |

### 失败示例

```json
{ "code": 400, "message": "请上传音频文件", "data": null }
```

```json
{ "code": 500, "message": "语音识别失败：讯飞识别失败（code=10165）：invalid handle", "data": null }
```

| code | 场景 |
|------|------|
| 400 | 参数错误：未上传文件、格式/语种不支持、音频内容为空 |
| 500 | 识别失败：讯飞返回错误、连接异常、识别超时（默认 60s） |

### 讯飞常见错误码（包含在 500 的 message 中）

| 讯飞 code | 含义 | 常见原因 |
|-----------|------|----------|
| 10105 | illegal access | appid 与 apiKey/apiSecret 不匹配 |
| 10114 | session timeout | 会话超时，音频过长或发送过慢 |
| 10163 | param invalid | business 参数非法 |
| 10165 | invalid handle | 首帧未带参数或帧顺序错误 |
| 11200 | auth no license | 账号无该能力授权或已过期 |
| 11201 | auth no enough license | 当日调用量超限 |

## 3. 调用示例

### curl

```bash
# mp3（自动按扩展名识别格式）
curl -X POST http://localhost:8080/api/asr/recognize \
  -F "file=@./test.mp3"

# 裸 pcm + 英文识别
curl -X POST http://localhost:8080/api/asr/recognize \
  -F "file=@./test.pcm" \
  -F "format=pcm" \
  -F "language=en_us"
```

### 小程序（uni-app）

```ts
// 录音：固定 16k 采样率、单声道
const recorder = uni.getRecorderManager();
recorder.start({
  format: 'mp3',
  sampleRate: 16000,
  numberOfChannels: 1,
});

recorder.onStop((res) => {
  uni.uploadFile({
    url: 'http://localhost:8080/api/asr/recognize',
    filePath: res.tempFilePath,
    name: 'file',
    formData: { language: 'zh_cn' },
    success: (uploadRes) => {
      const result = JSON.parse(uploadRes.data);
      if (result.code === 200) {
        console.log('识别结果：', result.data.text);
      } else {
        uni.showToast({ title: result.message, icon: 'none' });
      }
    },
  });
});
```

## 4. 服务端配置

`src/main/resources/application.yaml`：

```yaml
xfyun:
  asr:
    host-url: wss://iat-api.xfyun.cn/v2/iat   # 讯飞听写 WebSocket 地址
    appid: f286e12b                            # 讯飞开放平台应用 APPID
    api-key: ce45a1596fbc1b35a34d82f4cd83d3cb  # APIKey
    api-secret: NzYwMTUyZTMyYzc0MzQyNzE1OGRiYTBm # APISecret
    frame-interval-ms: 40                      # 每帧(1280B)发送间隔，官方建议 40ms
    timeout-seconds: 60                        # 整体识别超时
```

**关于响应耗时**：讯飞协议要求分帧模拟实时音频流上传，按官方建议的 40ms/帧，接口耗时约等于「音频时长 + 识别尾延迟」（10 秒音频约 10~11 秒返回）。若希望更快返回，可将 `frame-interval-ms` 调小（如 10），实测引擎可接受快于实时的上传速度。

## 5. 实现说明（后端）

| 文件 | 职责 |
|------|------|
| `controller/AsrController.java` | `POST /api/asr/recognize`，参数校验、格式推断、异常转 `Result.fail` |
| `service/AsrService.java` | 服务接口 |
| `service/impl/AsrServiceImpl.java` | 讯飞 IAT WebSocket 协议：HMAC-SHA256 签名握手、分帧上传（首帧带参数、尾帧 status=2）、结果帧拼接 |
| `dto/asr/AsrResponseDTO.java` | 响应 DTO（`text`、`sid`） |
| `config/SecurityConfig.java` | 放行 `/api/asr/**` |

识别流程：

```
客户端 ──multipart──▶ AsrController
                        │
                        ▼
                  AsrServiceImpl
                        │ 1. 生成 HMAC-SHA256 签名 URL，建立 WebSocket 连接
                        │ 2. 音频按 1280 字节/帧上传（首帧 status=0 带参数，
                        │    中间帧 status=1，尾帧 status=2）
                        │ 3. 监听结果帧，拼接 data.result.ws[].cw[].w
                        │ 4. data.status=2 → 识别完成，返回全文
                        ▼
                 讯飞 IAT WebSocket (wss://iat-api.xfyun.cn/v2/iat)
```
